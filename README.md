# BB-8
Consensus caller for R2C2 concatemeric reads - adapted directly from C3POa by R-Volden and C-Vollmers. Please view their original repository and associated publications:
- [C3POa repository](https://github.com/christopher-vollmers/C3POa)
- [Publication: Improving nanopore read accuracy with the R2C2 method enables the sequencing of highly multiplexed full-length single-cell cDNA](https://doi.org/10.1073/pnas.1806447115)

BB-8 is a modifed version of C3POa to better suit cfDNA sequencing using the R2C2 method. To view changes please see the ChangeLog

## Dependencies
BB-8 dependencies can be be installed using the setup script.

- Python3
- NumPy
- SciPy
- mappy
- edlib
- networkx
- pyaboa
- Cython
- wheel
- setuptools
- conk
- blat executable

To fetch and build dependencies, use setup.sh.
setup.sh will download and make the packages that you need to run C3POa.

```
chmod +x setup.sh
./setup.sh
```

## Usage
BB-8 usage is very similar to C3POa. After resolving all of the dependencies, you can run BB-8 with python.
```
python3 BB-8.py -r /path/to/reads.fastq -o /path/to/output/ -s UMI_Splints.fasta
python3 BB-8_postprocessing.py -i path/to/where/C3POa/outputs/data/ -a adapter.fasta -x sampleSheet
```
Note that BB-8_postprocessing.py takes the output folder of BB-8.py as input

## BB-8
BB-8 can be run on a fastq file or a directory containing multiple fastq files.
BB-8 has a --resume option that will look for a bb8.log file in your output direcotry. It will look into the bb8.log file in the output directory to determine if it was run previously on same of the fastqs in the input directory and then skip those.


### Command-line Options

Command-line Options:
```
  --reads, -r
      FASTQ file containing long R2C2 reads, or a directory containing multiple FASTQ files.

  --splint_file, -s
      Path to the splint FASTA file.

  --out_path, -o
      Output directory for all generated files.
      Defaults to the current directory. (optional)

  --lencutoff, -l
      Minimum length cutoff for raw sequences.
      Sequences shorter than this are excluded.
      Default: 1000

  --mdistcutoff, -d
      Median distance cutoff for consensus sequences.
      Sequences shorter than this are excluded.
      Default: 500

  --numThreads, -n
      Number of threads to use for multiprocessing.
      Default: `

  --compress_output, -co
      Compress (gzip) both consensus FASTA and subread FASTQ output files.

  --resume, -u
      Resume processing by checking for a processed.log file in the output directory.
      Previously processed reads will be skipped and outputs appended.

  --version, -v
      Print the C3POa version.
```

## BB-8_postprocessing
Trims and reorients consensus sequences generated by BB-8.py to 5'->3' direction. If given a fasta of oligo dT indexes, it will also demultiplex the reads by index.
Command-line Options:
```
  --input_folder, -i
      Input and output directory.
      Must be the same output directory used by C3POa.py.

  --adapter_file, -a
      FASTA file containing adapter sequences (3′ and 5′).

  --samplesheet, -x
      Sample sheet with a header line indicating index locations.

  --undirectional, -u
      Treat cDNA molecules as undirectional.
      By default, molecules are assumed directional, requiring two sequences
      named "3Prime_adapter" and "5Prime_adapter" in the adapter file.
      With this flag, only one sequence named "Adapter" is expected.

  --threads, -n
      Number of threads to use for multiprocessing.
      Default: 1

  --groupSize, -g
      Number of reads processed per thread per iteration.
      Default: 10000

  --maxDist, -M
      Maximum edit distance allowed between a read and the best-matching index
      in the sample sheet to be considered a match.
      Default: 2

  --minDist, -m
      Minimum difference in edit distance between the best and second-best
      matching index required to assign a read.
      Default: 1

  --blatThreads, -bt
      Chunk BLAT processing across threads instead of by groupSize (faster).

  --skip_trimming, -s
      Demultiplex an already trimmed dataset (skip trimming step).

  --compress_output, -co
      Compress (gzip) both consensus FASTA and subread FASTQ output files.

  --version, -v
      Print the program version.
```
## Sample sheet format
The sample sheet is a tab delimited file with one line per Splint/Index combination and what sample they are associated with. (Multiple lines with different Splint/Index combinations can point to the same sample.)

The header line tells the script how to demultiplex so formatting that right is important. There are two types of indexes:

If your reads are directional, meaning you had a 5prime and 3prime adapter you can use 5prime and 3prime indexes which the program expects to find at the respective ends of the read. You can have an arbitrary number of those but we have only tested one 5 and one 3 index. This is how your header line and first sample lines would look:
```
Name	Splint	5.0:16.AC.NNNNNNNN.TC	3.0:16.GT.NNNNNNNN.AC
Sample1	UMI_Splint1	ATATATAT	CGCGCGCG
Sample1	UMI_Splint3	CGATAGTG	ATCGAGTA
Sample2	UMI_Splint1	TGGTGGAT	TAGGACTA	
```
BB-8_postprocessing will look at the first 16 bases of the reads 5prime end for the first index which is 8 variable bases (N) flanked by AC and TC on either side. It will then look at the first 16 bases of the 3prime end for the second index which 8 varibale bases flanked by GT and AC on either side. Pick a range (e.g. "0:16" that will surely contain the index. Pick flanking bases based on sequence context of your library adapters. You can use Ns, and other IUPAC wildcard bases for both flanking and index positions.

If your reads are undirectional and you used the -u flag (for example regular Smartseq2 cDNA) but you used an index in either the TSO or oligodT primer, you can use an 'E' (for either) index. You can only use one E index and they are not compatible with 5prime and 3prime indexes. This is how your header line and first sample lines would look:

```
Name    Splint  E5.0:16.AC.NNNNNNNN.TC
Sample1 UMI_Splint1     ATATATAT
Sample1 UMI_Splint3     CGATAGTG
Sample2 UMI_Splint1     TGGTGGAT
```
The script will check both sides of the read for the index. The number (5 or 3) following the 'E' tells the script where the index should be (5' or 3') and the script will reorient the read based on where it finds the index so the read is 5'->3' in the output fasta.

Generally, the indexes you supply in the samplesheet should be in the orientation as they would appear in the PCR primers you amplified your library with. If you run into any trouble here, just include all orientations of the indexes and see which ones get all the reads.
